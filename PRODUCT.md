# Product

## Register

product

## Users

The primary users are Chinese learners, interviewers, and project reviewers who need to inspect a working RAG Agent quickly. They use the interface to ask questions about Cyberpunk: Edgerunners, Cyberpunk 2077, related music, worldbuilding, characters, plot interpretation, and official collaboration content.

## Product Purpose

Cyberpunk RAG Agent is a Streamlit-based knowledge assistant that demonstrates a complete RAG workflow: local knowledge ingestion, hybrid retrieval, reranking, parent-child context expansion, tool calling, source references, and Ragas evaluation. Success means the user can ask a domain question, receive a clear Chinese answer, see supporting sources, and trust that the project is more than a hard-coded demo.

## Brand Personality

Clear, technical, calm.

The product should feel like a focused knowledge console rather than a fan site or marketing page. It should communicate reliability, traceability, and interview-ready engineering maturity.

## Anti-references

Avoid decorative cyberpunk clichés, heavy neon, dark sci-fi dashboards, noisy gradients, oversized hero sections, card-heavy landing-page composition, and inconsistent button styles. The UI should not look like a generic AI-generated SaaS mockup or a flashy game fansite.

## Design Principles

1. Keep the conversation first: the chat surface is the primary workflow, so controls should support asking, reading, and checking sources without visual noise.
2. Make trust visible: references, knowledge status, and retrieval-related actions should be easy to find and visually stable.
3. Use product restraint: prefer a white-blue, readable, high-contrast tool interface over dramatic cyberpunk styling.
4. Preserve demo clarity: a reviewer should understand what the project does within seconds, without reading a marketing page.
5. Treat maintenance actions carefully: destructive or expensive actions such as clearing chat or rebuilding the knowledge base should be distinct from ordinary question shortcuts.

## Accessibility & Inclusion

Aim for readable Chinese text, clear focus states, adequate contrast for body text and placeholders, keyboard-friendly controls, and reduced-motion support. Motion should be subtle and state-driven, not decorative.
